# Commands

> **Implementation Status (amended `step45.md`)**: this page is an early, never-filled-in outline — the space-separated syntax below (`status away`, `team watch`, `github prs`, ...) was never built, and none of `team`/`notify`/`dashboard` exist as commands at all. The real in-TUI command bar (`:` to open, `crates/ui/src/keyboard.rs`, `step9.md`/`step13.md`) uses slash-commands instead: `/send <channel> <text>` (Slack), `/away`/`/active`/`/offline`/`/meeting`/`/lunch [text]` (presence), `/pomodoro start|pause|reset [work] [break]`, `/calendar-range <hours>`, the picker overlays' own selection-management (`step41.md`) — `/slack-watch #channel [...]`, `/repo-watch owner/repo [...]`, `/calendar-rename <old> <new>`, `/calendar-remove <label>` — `/read-all` (`step44.md`, marks every currently-unread notification read in one shot, the command-line equivalent of pressing `Enter` on each Notification-panel row individually), and `/sync` (`step46.md`, gives `Command::SyncAllAdapters` — declared since Phase 3 but never wired to anything — a real implementation: runs one poll cycle immediately on every connected adapter, out-of-band from each one's own interval timer, instead of waiting for the next scheduled tick). `Tab` autocompletes command heads, `/send`/`/slack-watch`'s channel argument(s), `/repo-watch`'s repository argument(s), and `/calendar-rename`/`/calendar-remove`'s target-calendar argument (`step45.md`) — all matched case-insensitively by prefix against whatever the corresponding picker last fetched. **Deliberately excluded**: credential entry (`Ctrl+S`/`Ctrl+G`/`Ctrl+L`) has no command equivalent and never will while the command bar's `history` stays plain text — a `/connect ... <token>` command would put a live secret there, defeating the masked-input overlays that exist specifically to avoid that (`step41.md`'s Context). Separately, Phase 15 (`step15.md`) added a **CLI-level** (not in-TUI) command surface — `termws slack-send <channel> <text>`, `termws set-presence <status> [text]`, `termws status` — for scripting against an already-running instance from another terminal; see `step15.md` and the README's usage section for the real, current command syntax on both surfaces.
