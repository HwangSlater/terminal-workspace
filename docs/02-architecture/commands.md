# Commands

> **Implementation Status (amended `step41.md`)**: this page is an early, never-filled-in outline — the space-separated syntax below (`status away`, `team watch`, `github prs`, ...) was never built, and none of `team`/`notify`/`dashboard` exist as commands at all. The real in-TUI command bar (`:` to open, `crates/ui/src/keyboard.rs`, `step9.md`/`step13.md`) uses slash-commands instead: `/send <channel> <text>` (Slack), `/away`/`/active`/`/offline`/`/meeting`/`/lunch [text]` (presence), `/pomodoro start|pause|reset [work] [break]`, `/calendar-range <hours>`, and, since `step41.md`, the picker overlays' own selection-management: `/slack-watch #channel [...]`, `/repo-watch owner/repo [...]`, `/calendar-rename <old> <new>`, `/calendar-remove <label>` — all with `Tab` autocomplete on command heads and `/send`'s channel argument. **Deliberately excluded**: credential entry (`Ctrl+S`/`Ctrl+G`/`Ctrl+L`) has no command equivalent and never will while the command bar's `history` stays plain text — a `/connect ... <token>` command would put a live secret there, defeating the masked-input overlays that exist specifically to avoid that (`step41.md`'s Context). Separately, Phase 15 (`step15.md`) added a **CLI-level** (not in-TUI) command surface — `termws slack-send <channel> <text>`, `termws set-presence <status> [text]`, `termws status` — for scripting against an already-running instance from another terminal; see `step15.md` and the README's usage section for the real, current command syntax on both surfaces.
