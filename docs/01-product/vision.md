# Vision

## Goal
Keep developers inside the terminal.

## Guiding Principle

> A user should never have to study the platform to use it.

Installation, configuration, databases, toolchains, and migrations are all complexity the platform absorbs internally. A user runs `tw` in a terminal and is immediately working. See `product-requirements.md` §2 for how this splits into a **User Experience** principle (Zero Setup / One Command Startup) and a separate **Contributor Experience** principle (One-Time Setup) — they are not the same audience and should not be held to the same bar.

## Core Principles
- Terminal first
- Event driven
- Plugin based
- Fast startup
- Minimal resource usage

## Non Goals
- Replace Slack
- Replace GitHub
- Become a terminal emulator
